1. Development Steps
First, I created a Splash Screen that automatically redirects the user to the Login screen after a few seconds.
I set up the Express.js backend, connected it with a live MongoDB database, and configured Cloudinary for storing uploaded images.
After that, I developed the Login and Signup screens and connected them with the backend APIs.
I created the Create Story screen where users can select an image from their phone gallery and upload it.
To keep the code organized and easier to manage, I separated the Create Story screen into smaller reusable components such as TopControls and BottomControls.
I then developed the Story Viewer screen, which displays uploaded stories in full-screen mode using ImageBackground.
Finally, I implemented story navigation so users can move between stories by tapping on the screen, and the application automatically opens the next user's stories when the current stories finish.

3. Problems Faced
The Splash Screen navigation did not work correctly during the initial implementation.
Image uploads failed after backend deployment because local storage was not persistent on Vercel.
Understanding how frontend forms communicate with backend APIs was challenging at first.
Passing authentication tokens and handling protected routes was confusing.
Implementing story navigation without visible buttons required additional research and testing.
Managing story indexes and moving between multiple users' stories was one of the most difficult parts of the project because several edge cases had to be handled correctly.

4. Solutions
I fixed the Splash Screen issue by correctly placing setTimeout inside a useEffect hook.
To solve image upload issues, I replaced local multer storage with Cloudinary-based storage.
I improved my understanding of API communication by practicing Axios requests and testing endpoints.
I learned how to send authorization headers properly when accessing protected routes.
For story navigation, I used the tap position on the screen to determine whether the user wanted to move forward or backward.
I carefully adjusted the story navigation logic to handle transitions between different users' story groups.

5. Learning Reflection
What I Learned: This project helped me understand how a React Native frontend communicates with a live backend and database. I learned how to use Axios for API requests, handle user authentication, upload images using FormData, and organize code into reusable components. I also gained practical experience with Cloudinary and backend deployment.
What was difficult: The most challenging part was implementing the Story Viewer logic. Managing multiple story groups, tracking indexes, and ensuring smooth navigation between stories required a lot of testing and debugging. I had to try different conditions several times before the navigation worked correctly. Understanding authentication flow and image uploads also took time because I had not worked extensively with these concepts before.
